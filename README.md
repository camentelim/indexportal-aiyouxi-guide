# IndexPortal-Aiyouxi-Guide

本仓库用于归档和发布多个独立 HTML 页面，提供对各类网页内容的整理与索引。不指向任何特定域名或网站，仅作为通用页面集合的存储与展示平台。

## 项目简介

本项目旨在收集、整理并维护一组独立的 HTML 页面，涵盖不同主题与功能。所有页面均为静态资源，可直接在浏览器中打开，无需后端服务支持。仓库本身不包含任何推广、营销或诱导点击的内容，仅作为技术性存档与分发用途。

## 目录说明

```
.
├── README.md          # 本文件
├── index.html         # 主入口页面（可选）
├── pages/             # 存放所有独立 HTML 页面
│   ├── page1.html
│   ├── page2.html
│   └── ...
└── assets/            # 公共资源（样式、脚本、图片等）
    ├── css/
    ├── js/
    └── images/
```

- **pages/**：存放所有归档的 HTML 页面文件，每个页面独立且互不依赖。
- **assets/**：存放页面共用的静态资源，如样式表、JavaScript 脚本和图片等。
- **index.html**：可选的入口页面，用于快速导航到各归档页面。

## 页面归档说明

本仓库中的 HTML 页面均以独立文件形式存在，每个文件代表一个完整的网页。页面内容可能涵盖信息展示、工具、参考文档等，但不会包含任何具体域名、网址或商业推广信息。

- 页面命名遵循简洁、描述性的原则，便于识别。
- 所有页面均采用标准 HTML5 规范编写，兼容主流浏览器。
- 页面之间不共享状态或数据，确保独立运行。

## 维护说明

- 本仓库由维护者不定期更新，新增页面或修正现有页面内容。
- 欢迎提交 Issue 或 Pull Request 建议改进，但请勿添加任何形式的广告或推广内容。
- 所有提交的页面需遵循静态 HTML 规范，不依赖外部服务或动态后端。

## 使用方式

1. 克隆或下载本仓库到本地。
2. 直接在浏览器中打开 `pages/` 目录下的任意 HTML 文件。
3. 若需要整体浏览，可访问 `index.html`（如果存在）。

## 贡献指南

- 提交新页面时，请将文件放入 `pages/` 目录，并在必要时更新 `index.html` 中的链接。
- 确保页面不包含外部域名引用、跟踪代码或任何形式的商业推广。
- 保持页面简洁、轻量，避免过度依赖外部资源。
